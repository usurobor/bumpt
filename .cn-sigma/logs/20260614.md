### Learnings worth canonicalizing at home

1. **Neon serverless HTTP driver can't reach plain Postgres in CI.** Run it through
   a local Neon HTTP proxy (`ghcr.io/timowilhelm/local-neon-http-proxy`,
   `db.localtest.me:4444`); gate the driver's `fetchEndpoint` behind an env flag so
   production is untouched.
2. **`neon()` validates the connection-string *format* at import.** An offline
   build (`DATABASE_URL` unset) needs a well-formed-but-unroutable fallback, not a
   junk placeholder, or `next build` throws everywhere except where the env is set.
3. **Assert DB state via `psql`, not by re-reading rendered pages.** A
   `force-dynamic` page read back stale under the proxy even though the row was
   committed; render/proxy caching makes HTML a bad oracle. Query Postgres directly.
4. **Annotated tags in CI need a git identity** (`git config user.name/email`).
5. **Notification pattern generalizes:** green = headline + aggregate; red =
   per-check breakdown + aggregate; one shared sender; no-op without secrets.

Recommend home consider a cnos-level convention for the notification practice (as
with the activation-log convention) and the three-branch CI/CD model.
